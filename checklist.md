## Portfolio Website Checklist

Use this as a simple TODO list for your assignment and portfolio.

### Structure & Pages
- [ ] Confirm you have **5–10 HTML pages** (`index`, `about`, `projects`, `blog`, post page, `contact`, `map`, `policy`, etc.).
- [ ] Make sure the **navigation bar is consistent** across all pages.
- [ ] Verify all **internal links** work (menu, buttons, “More projects”, blog links, etc.).

### Content
- [ ] Replace **placeholder text** on the home page (hero, about, skills, experience, education, blog preview, contact CTA).
- [ ] Update **project cards** with real projects (title, short description, tech stack, link to details).
- [ ] Fill in **blog content** (at least one real post).
- [ ] Update **contact information** (email, phone, address) everywhere it appears.

### Media & Iframe
- [ ] Add at least **5 images** with meaningful `alt` text.
- [ ] Add/verify at least **one `<figure>` + `<figcaption>`**.
- [ ] Ensure there is at least **one `<iframe>`** (e.g. Google Maps / YouTube) on the `map.html` or relevant page.

### Form Requirements
- [ ] On the contact (or another) page, include a **full-featured form** with:
  - [ ] `text`, `email`, `tel`, `password`, `number`, `date`, `range`, `color`, `file`
  - [ ] `radio`, `checkbox`, `select` (with `optgroup`), `textarea`, `datalist`, `hidden`
  - [ ] `submit` and `reset` buttons
- [ ] Use **`label` for all inputs**, and group with at least one `fieldset` + `legend`.
- [ ] Add **`required`/`pattern`/`min`/`max`/`step`** where appropriate.
- [ ] Show **friendly error messages** (native HTML messages are ok, JS enhancement optional).

### CSS & Responsive Design
- [ ] Keep styles in the **external CSS file** (`css/style.css`).
- [ ] Ensure at least **one responsive breakpoint** (~768px) looks good (nav, hero, grids).
- [ ] Check **hover/focus states** are visible on links, buttons, form controls.
- [ ] Use consistent **color scheme** and **typography** across pages.

### Accessibility
- [ ] Check **heading order** (`h1` → `h2` → `h3`…) on each page.
- [ ] Confirm **contrast** and **font size ≥ 16px** for body text.
- [ ] Test **keyboard navigation** (Tab/Shift+Tab) and visible focus outlines.
- [ ] Make sure all images have useful **`alt` text**.

### Optimization & Quality
- [ ] Use reasonably sized / optimized image files (JPG/PNG/WebP).
- [ ] Ensure each page has a **good `<title>`** and **`meta description`**.
- [ ] Keep folder structure clean (`assets/`, `css/`, `img/`, `js/`, `docs/`, etc.).
- [ ] Run pages through **W3C HTML/CSS validators** and fix major errors.

### Deployment & Docs
- [ ] Deploy to **GitHub Pages or Netlify** and verify all pages load correctly.
- [ ] Keep the **GitHub repo up to date** with clear commit messages.
- [ ] Update **README** with site map, features, how to run, and credits.
- [ ] Keep `docs/AI_usage_logs.md` updated whenever you use AI help.
